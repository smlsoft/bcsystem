# GitHub Copilot Rules for DedeCashier

## 🚫 ห้ามเด็ดขาด

1. **ห้ามสร้าง Markdown files** (.md) โดยไม่ได้รับอนุญาต
2. **ห้ามแก้ Business Logic** ที่กระทบการทำงานเดิม
3. **ห้าม Blocking Operations** ที่ทำให้ UI สะดุด
4. **ห้ามแก้โค้ดโดยไม่ถามก่อน**

## ✅ ต้องทำ

1. **ถามก่อนทุกครั้ง** ก่อนแก้ไขโค้ด
2. **ค้นหาข้อมูลจาก Internet** ก่อนตัดสินใจ
3. **วิเคราะห์จุดอ่อน** ของโค้ดทุกครั้ง
4. **Optimize Performance** โดยไม่เปลี่ยน output
5. **Improve UX/UI** ให้ลื่นไหลและดูสวยงาม

## 📋 Checklist ก่อนแก้โค้ด

- [ ] ค้นหา best practices จาก documentation
- [ ] วิเคราะห์จุดอ่อนของโค้ดปัจจุบัน
- [ ] ถามและแสดงแผนการแก้ไข
- [ ] รอคำตอบก่อนดำเนินการ
- [ ] แสดง Before/After comparison
- [ ] อธิบายผลกระทบและความเสี่ยง
- [ ] รับประกันว่าทำงานเหมือนเดิม

## 🎯 เป้าหมายหลัก

**Performance First, UX/UI Excellence, Business Logic Sacred**

### หลักการทำงาน:
1. 🔍 **วิเคราะห์** - หาจุดอ่อนก่อนเสมอ
2. 🌐 **ค้นคว้า** - หา best practices จาก internet
3. ❓ **ถาม** - แสดงแผนและรอคำตอบ
4. ⚡ **แก้ไข** - Optimize โดยไม่กระทบ logic
5. 🎨 **ปรับปรุง** - ทำให้ดูสวยและลื่นไหล
6. ✅ **ตรวจสอบ** - รับประกันคุณภาพ

ดูรายละเอียดเพิ่มเติมใน `.github/copilot-instructions.md`
